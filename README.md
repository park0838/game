# 🎨 실시간 협력 드로잉 게임

링크만 공유하면 여러 명이 동시에 하나의 캔버스에 그림을 그릴 수 있는 웹 게임입니다!

## ✨ 특징

- 🚀 **서버 없이 동작**: WebRTC P2P 기술로 서버 비용 0원
- 🔗 **링크 공유로 즉시 참가**: 복잡한 회원가입 없이 링크만 공유
- ⚡ **초저지연 실시간**: P2P 직접 연결로 빠른 반응속도
- 🎨 **다양한 색상과 도구**: 8가지 색상, 굵기 조절, 지우개
- 👥 **동시 다중 접속**: 여러 명이 동시에 그리기 가능
- 👆 **실시간 커서 표시**: 다른 사람의 마우스 위치 확인
- 📱 **모바일 지원**: 터치스크린에서도 작동

## 🚀 실행 방법

### 1. 로컬에서 실행

```bash
# 현재 디렉토리에서 간단한 서버 실행
python3 -m http.server 8000

# 또는 Node.js 사용
npx http-server -p 8000
```

### 2. 브라우저에서 열기

```
http://localhost:8000
```

### 3. 게임 시작

**방 만들기:**
1. "새 방 만들기" 버튼 클릭
2. 생성된 방 코드 확인
3. 📋 버튼을 클릭하여 링크 복사
4. 친구들에게 링크 공유!

**방 참가하기:**
1. 친구로부터 받은 링크 클릭 (자동 참가)
2. 또는 방 코드를 입력하고 "방 참가" 클릭

## 🎮 사용 방법

- **그리기**: 마우스를 드래그하여 그림 그리기
- **색상 변경**: 원하는 색상 버튼 클릭
- **굵기 조절**: 슬라이더로 브러시 크기 변경
- **지우개**: 지우개 버튼으로 지우기 모드
- **전체 지우기**: 🗑️ 버튼으로 캔버스 초기화

## 🔧 기술 스택

- **프론트엔드**: Vanilla JavaScript, HTML5 Canvas
- **P2P 연결**: WebRTC + PeerJS
- **스타일링**: CSS3 (Flexbox, Gradient)
- **STUN 서버**: Google Public STUN

## 📝 작동 원리

1. **방 생성**: 호스트가 PeerJS를 통해 고유 ID 생성
2. **링크 공유**: 생성된 ID가 포함된 URL 공유
3. **P2P 연결**: 참가자가 링크 클릭 시 WebRTC로 직접 연결
4. **데이터 동기화**: 드로잉 데이터를 P2P로 실시간 전송
5. **커서 공유**: 마우스 위치도 실시간으로 공유하여 협업 가능

## 🐛 문제 해결

**연결이 안 될 때:**
- 방화벽 설정 확인
- HTTPS에서 실행 (일부 브라우저는 HTTP에서 WebRTC 차단)
- 다른 브라우저 시도 (Chrome, Firefox 권장)

**그림이 느릴 때:**
- 참가자 수가 너무 많으면 느려질 수 있음 (권장: 5명 이하)
- 인터넷 연결 상태 확인
